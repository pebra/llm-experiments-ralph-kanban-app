## Codebase Patterns
- Use `bun:sqlite` Database for all persistence
- Database defaults to `./kanban.db` file; use `setDbPath(":memory:")` for tests
- API endpoints use Bun.serve() routes pattern with `Response.json()`
- Frontend fetches data from `/api/columns` endpoint on mount
- Tests use `bun:test` with `beforeEach` calling `setDbPath(":memory:")` for isolation
- HTML5 native drag-and-drop API used for kanban (no external DnD library)
- `refreshColumns()` pattern: after any mutation, re-fetch from `/api/columns` to sync UI
- API endpoints for task mutations: POST create, PATCH update, DELETE delete, PUT move
- `getDb()` exported from db.ts for ad-hoc queries in API handlers
- Column reorder uses position-swap approach (swap positions, don't shift)
- Drag data distinguished by prefix: `col-{id}` for columns, plain `{id}` for tasks
- All db operations wrap errors with context (e.g., "Failed to create task: {message}")
- `setDbPath()` closes existing connection and resets the db singleton
